## 2. เดโมทำงานยังไง (การคำนวณแดด/ร่ม 4 ขั้น)

1. **ตำแหน่งดวงอาทิตย์** — ฟังก์ชัน `sunPosition(date,lat,lon)` (อัลกอริทึม SunCalc เดียวกับ `MapView.jsx`) คืน azimuth + elevation คำนวณเอง ไม่พึ่ง lib/เน็ต
2. **ความสูงตึก (h)** — ลำดับความน่าเชื่อ: ความสูงวัดจริงจากดาวเทียม (`_src:"sat"`) → OSM `height`/`building:levels` → เดาจากชนิดอาคาร (ตาราง `TYPE_H`) → default 12 ม.
3. **ฉายเงาลงพื้น** — ต่อตึก: ทิศเงา = `azimuth+180°`, ยาวเงา = `h / tan(elevation)`; รูปเงา = Minkowski sum ของ footprint กับเวกเตอร์เงา (elevation ≤ 3° = ไม่ฉายเงา = ร่มทั้งย่าน/กลางคืน)
4. **% ร่มของเส้นทาง** — สุ่มจุดทุก ~10 ม. เช็คว่าจุดตกในเงาตึกไหม (point-in-Minkowski-sum) → เขียว=ร่ม เหลือง=แดด → `% ร่ม = จุดในเงา / จุดทั้งหมด` อัปเดตสดตามสไลเดอร์เวลา

> ยังไม่คิด: ตึกบังกันเอง (คิดเงาต่อตึกอิสระ) · ตรงนี้เป็น prototype ของ "เงาตึก" อย่างเดียว

## 3. ความสูงตึกจริง (สำคัญ — จุดที่เพิ่งแก้ให้แม่น)

- **เดิม** เดา default 12 ม. สำหรับตึกที่ OSM ไม่มี tag (~65%) → เงาเพี้ยน
- **ตอนนี้** ใช้ **Google Open Buildings 2.5D Temporal V1** (band `building_height`, Sentinel-2, ~4 ม., ปี 2016–2023, ครอบคลุมไทย) เป็นความสูง "วัดจริง"
- ดึงผ่าน **Google Earth Engine** (บัญชีฟรี) — เบราว์เซอร์/แซนด์บ็อกซ์ดึงสดไม่ได้ ต้อง export ครั้งเดียว
- ผลลัพธ์ปัจจุบัน: 374 ตึก, ความสูง median 16.6 ม. สูงสุด 97 ม. (ตึกสูงแถวสยาม)
- เดโมโหลดไฟล์นี้อัตโนมัติ ตึกวัดจริง = **สีเขียวเทอควอยซ์** + แถบสรุปโชว์ % ที่วัดจริง

### วิธีดึงความสูงพื้นที่/ปีใหม่
1. เปิด https://code.earthengine.google.com → วาง `scripts/extract_heights_gee.js` → แก้ `aoi` (bbox) / `year` → Run
2. แท็บ Tasks → Run งาน export → ได้ GeoJSON ใน Google Drive
3. โหลดเข้าเดโมด้วยปุ่ม "📐 ความสูงจริง" หรือแทนไฟล์ `walkbkk_heights_2023.geojson`/`.js`

> **แหล่งอื่นที่เช็กแล้ว:** Overture Maps = height ในกรุงเทพแทบว่าง ไม่คุ้ม · LiDAR กรุงเทพยังไม่มี open data

## 4. เส้นทางเดโม

- ปัจจุบัน: **MBK Center → BTS สยาม ผ่าน Skywalk** — **ตรงกับดีไซน์ Figma "BDI-Siam"** (frame `route` / vector `skywalk_patumwan`)
- เส้นหลัก = `SKYWALK_ROUTE` (hardcode ในไฟล์ พร้อมคอมเมนต์ทีละ waypoint): MBK ชั้น 2 โซน A (ฝั่ง Don Don Donki) → ออกทางเชื่อม Skywalk → **วนตามวงแหวนแยกปทุมวัน (ฝั่งตะวันออก)** → Siam Discovery → Siam Center → ทางเชื่อม Paragon → BTS สยาม (~550 ม.)
- **สัญลักษณ์บนแผนที่** (แบบเดียวกับ Figma): พื้นโครงสร้าง Skywalk วาดเป็น polygon จริง (`SKY_RING` วงแหวน r 42 ม. รอบแยก + `SKY_ARMS` แขนไปสนามกีฬาฯ/สยาม/พญาไท) · แถบคาดใต้เส้นทาง ฟ้า=บน Skywalk ม่วง=ในห้าง (`SKYWALK_ZONES`) · ป้าย DOM marker กำกับจุดสำคัญ (`SKYWALK_POIS`) — ซ่อนอัตโนมัติเมื่อสลับไปเส้นถนน OSM
- พิกัด georeference จาก footprint จริงใน `walkbkk_heights_2023.geojson` (MBK / Siam Center / Paragon) — **ไม่เรียก OSRM อัตโนมัติแล้ว** ปุ่ม "🚶 เส้นถนน (OSM)" ยังกดดูเส้นระดับถนนได้ และปุ่ม "🌉 เส้น Skywalk (Figma)" กลับมาเส้นหลัก
- แผนที่หมุนให้ทิศที่เดินชี้ขึ้นบนจอ (เหมือนโหมดนำทาง) · เปลี่ยนต้น/ปลายทางแก้ที่ `START_PT`/`END_PT`/`SKYWALK_ROUTE` ในไฟล์
- ปุ่มในเดโม: เส้น Skywalk (Figma) · เส้นถนน (OSM) · จัดมุม · มุมบน-ลง-ล่าง · โหลดตึก OSM · โหลดความสูงจริง
